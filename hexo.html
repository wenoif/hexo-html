部署到Gitee

  1. Gitee新建仓库

  2. 配置SSH

    本地生成：ssh-keygen -t rsa -C "yourname@gitee.com"

    查看公钥：cat ~/.ssh/id_rsa.pub

    公钥配置到Gitee

    测试链接：ssh -T git@gitee.com

    返回结果：Hi xxxx! You've successfully authenticated, but GITEE.COM does not provide shell access.

Github自动部署Hexo

利用GitHub Actions 实现自动部署 Hexo。两个仓库，一个仓库hexo-md存放源文件（可以设置成私有），一个仓库hexo-html存放前端，修改源文件后自动生成部署到html仓库。

步骤

  • 本地可以通过 hexo clean && hexo g && hexo d 生成运行上传hexo，在 .github/workflows/deploy.yml新建一个监听代码，然后上传到hexo-md仓库，当监听到仓库变化时候自动生成部署到hexo-html仓库。为了两个仓库的安全，需要在生成一对公钥私钥，私钥放到源文件仓库hexo-md中，公钥放到前端代码仓库hexo-html中。

  • 在Hexo配置文件中添加前端仓库地址：

  • 生成密钥对：ssh-keygen -t rsa -b 4096 -C "Hexo Deploy Key" -f github-deploy-key -N ""，生成两个文件：github-deploy-key （密钥）和 github-deploy-key.pub （公钥）

  • 在Github的两个仓库中配置密钥：

    • 存放源文件的hexo-md仓库配置私钥：Settings -> Secrets->New secret
      • 名称Name填写HEXO_DEPLOY_KEY（和下面GitHub Actions Workflow中的一致），
      • 内容Value是github-deploy-key 中的内容。
    • 存放前端代码的hexo-html仓库配置公钥：Settings -> Deploy keys -> Add deploy key
      • Title中输入HEXO_DEPLOY_PUB或者其他名称，
      • Key 中粘贴 github-deploy-key.pub 文件的内容。
    • 一定要勾选 Allow write access 来打开写权限。
