简要介绍

Windows Subsystem for Linux（简称WSL）windows下的Linux子系统，是一个在Windows 10上能够运行原生Linux二进制可执行文件（ELF格式）的兼容层。

官方安装指南:https://docs.microsoft.com/zh-cn/windows/wsl/install-win10

安装之前

安装之前需要启用WSL功能

方式一：使用GUI启用Windows功能

  1. 打开“开始”菜单，然后搜索“ 打开或关闭Windows功能”
  2. 选择适用于Linux的Windows子系统，点击确定
  3. 出现提示时重新启动计算机

方式二：使用PowerShell

  1. 以管理员身份打开PowerShell并运行：

  2. 出现提示时重新启动计算机

正式安装

安装官方安装指南的方法无法将 Linux 安装到系统盘之外的分区

若需要安装到系统盘之外的分区则可以使用手动下载安装指南

或者在安装之后进行迁移

容器安装

开启WSL，如果这个不会，请直接关闭本页面。

下载centos镜像文件，点我下载

安装chocolatey，如果不会请参考这里

使用chocolatey安装LxRunOffline
cinst LxRunOffline -y

使用LxRunOffline安装CentOS

LxRunOffline.exe install -n centos -d E:\ProgramData\CentOS -f E:\ProgramDatacentos-7-docker.tar.xz

其中 -d 后面是要安装到的目录； -f 后面是第二部下载的镜像存放位置；-n 用来指定名称

安装完成之后，使用LxRunOffine 来开启 Centos
LxRunOffline run -n centos

开启SSH

SSH分为“客户端openssh-client”和“服务端openssh-server“

  • 服务端：开放本机ssh服务

    安装服务端：sudo apt-get install openssh-server

    启动ssh：sudo /etc/init.d/ssh start

    关闭ssh：sudo /etc/init.d/ssh stop

    查看状态：service ssh status #* sshd is running 显示此内容则表示启动正常

  • 服务端配置：sudo vi /etc/ssh/sshd_config

    Port = 22

    ListenAddress 0.0.0.0 #指定监听的IP，默认即监听PC所有IP。

    PermitRootLogin no #如果你需要用 root 直接登录系统则此处改为 yes

    PasswordAuthentication no #将 no 改为 yes 表示使用帐号密码方式登录

  • 服务端启动报错

    • 报错：sshd: no hostkeys available -- exiting

      解决：ssh-keygen -A 然后重启

    • WSL CentOS报错“Failed to get D-Bus connection: Operation not permitted”解决方法“更换一下 systemctl 文件”

      源地址：“https://github.com/DDoSolitary/LxRunOffline/issues/88”

  • 客户端：ssh登陆其他机器

    安装客户端：sudo apt-get install openssh-client

    连接ssh：ssh user@127.0.0.1 user为192.168.0.100机器上的用户，需要输入密码。

    断开：exit

手动获取WSL IP及端口转发

每次用wsl2虚拟机都会分配一个ip，使用windows自带的端口转发命令netsh可以实现在获取了虚拟机ip之后转发至wsl2进而进行访问。

  • 获取IP：arp -a 或者进入WSL内获取(每次启动都会变，通常是172ip段，类型为“动态”)

  • 设置端口转发（需要管理员权限运行）

    netsh interface portproxy add v4tov4 listenport=【宿主机windows平台监听端口】 listenaddress=0.0.0.0 connectport=【wsl2平台监听端口】 connectaddress=【wsl2平台ip】

  • 查看转发端口

    netsh interface portproxy show all

  • 删除端口转发

    netsh interface portproxy delete v4tov4 listenport=【宿主机windows平台监听端口】 listenaddress=0.0.0.0

wsl

版权所有 (c) Microsoft Corporation。保留所有权利。

用法: wsl.exe [参数] [选项…] [命令行]

用于运行 Linux 二进制文件的参数:

 如果未提供命令行，wsl.exe 将启动默认的 shell。

 --exec, -e <命令行>
     执行指定的命令而不使用默认的 Linux shell。

 --
     按原样传递剩余的命令行。
 选项:
     --distribution, -d <分发版>
     运行指定的分发。

     --user, -u <用户名>
       以指定用户身份运行。

用于管理适用于 Linux 的 Windows 子系统的参数:

 --export <分发版> <文件名>
     将分发导出到 tar 文件。
     对于标准输出，文件名可以是 -。

 --import <分发版> <安装位置> <文件名> [选项]
     将指定的 tar 文件作为新分发进行导入。
     对于标准输入，文件名可以是 -。

     选项:
         --version <版本>
             指定用于新分发的版本。

 --list, -l [选项]
     列出分发。

     选项:
         --all
             列出所有分发，包括当前正在
             安装或卸载的分发。

         --running
             只列出当前正在运行的分发。

         --quiet, -q
             只显示分发名称。

         --verbose, -v
             显示有关所有分发的详细信息。

 --set-default, -s <分发版>
     将分发设置为默认值。

 --set-default-version <版本>
     更改新分发的默认安装版本。

 --set-version <分发版> <版本>
     更改指定分发的版本。

 --shutdown
     立即终止所有正在运行的分发和 WSL 2 轻型工具虚拟机。

 --terminate, -t <分发版>
     终止指定的分发。

 --unregister <分发版>
     注销分发。

 --help
     显示用法信息。
